# 万模 AI · 前端重构设计规范

> 最后更新：2026-06-07。本文是 Chatty / 万模AI 前端视觉与交互规范；当前工程为 Next.js 16 + React 19 + Tailwind v4 + Radix UI + Zustand + Lucide。
>
> **风格定调**：新中式 · 墨韵
> **目标设备**：桌面（≥1024px） / 平板（768–1023px） / 移动端（<768px），移动优先
> **品牌主张**：一念通达 · 万模为器
> **说明**：历史静态高保真稿已移除，本文保留为当前前端设计规范与页面清单。

## 一、为什么是"新中式 · 墨韵"

国产 AI 产品的同质化已经非常严重 —— 几乎清一色的深紫渐变 + 霓虹光晕 + 西式无衬线大字。Chatty / 万模AI 想做出辨识度，最稳妥的差异化路径是回到**东方审美的本体**：

1. **色彩克制**：以"月白 + 瓷青"为底，朱砂只作为印章和警示，黄金作为荣誉色。整个画面只允许一处朱砂高亮。
2. **形态有气**：用宋瓷的圆润、宣纸的肌理、水墨的远山做装饰，避免硅谷常见的几何渐变光斑。
3. **字体双修**：思源宋体承担品牌与小标题，思源黑体承担正文 UI —— 制造"古今对话"的层次感。
4. **印章为礼**：朱砂印章作为锚点性视觉元素，仅在欢迎区、万模卡、关键成就出现，**绝不泛滥**。

> 参考对照：钉钉的工具感 + 小红书的温度 + 苹果 SF 的克制 + 故宫文创的雅 — 取其中庸。

**铁律**：
- 永远不用纯白 `#FFF`、纯黑 `#000`、Material 紫（`#6366F1`/`#A855F7`）—— 这三个是当前界面的"硅谷味"主犯。
- 一屏只允许 **一处朱砂**（印章 或 CTA，二选一），避免红色泛滥。
- 渐变只用同色系：瓷青 600 → 瓷青 800；月白 100 → 米色 200。**禁用对撞色渐变**。

font-family-serif: "Noto Serif SC","Songti SC","SimSun",serif;

| 按钮 / 链接 | 思源黑体 | 500 | 14px |
| 数字 / ID / 代码 | JetBrains Mono | 400–500 | 12–32px |

| Token | 值 | 场景 |
|---|---|---|

| radius-md | 12px | 输入框、普通按钮 |

**铁律**：阴影偏暖（基于墨色 + 棕调），**绝不使用纯黑或紫色投影**。

### 2.4 间距栅格

- 基础单位：4px
- 常用：4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 48 / 64
- 桌面内容最大宽度：`1400px`
- 移动端边距：左右 16px，安全区 `env(safe-area-inset-*)` 全启用

## 三、布局与响应式断点

| 断点 | 区间 | 主体策略 |
|---|---|---|
| `xs` | <480px | 单列、卡片满宽、底部 Tab |
| `sm` | 480–767px | 单列、横向滚动 chip |
| `md` | 768–1023px | 双列、参数面板抽屉式 |
| `lg` | 1024–1279px | 侧栏 +主区两栏 |
| `xl` | ≥1280px | 侧栏 + 主区 + 右侧栏（绘图三栏） |

**关键布局**：
- **对话页**：`260px` 侧栏 + 主区流动；移动端侧栏收起为左上 ≡ + 抽屉
- **绘图页**：`260px + flex + 320px`；<1024px 时右栏折叠为「⚙ 参数」按钮 → 底部抽屉
- **后台**：`220px` 深色侧栏 + 主区；移动端只保留概览（只读）
- **登录**：桌面双栏（左意境右表单），<768px 单列叠加（顶部 banner 缩小 + 表单全宽）

## 四、组件库（Component Inventory）

### 4.1 基础（沿用 shadcn/ui，重新主题化）

| 组件 | 关键改动 |
|---|---|
| Button (Primary) | 瓷青 600→500 渐变 + 12px 圆角 + ink 阴影 |
| Button (Ghost) | 透明 + 墨色边框 1px + hover 米色背景 |
| Input | 月白 50 底 + 墨 700/10 边框，无 focus 蓝光，改为瓷青 500 1.5px |
| Card | 月白 50 + 20px 圆角 + paper 阴影 |
| Tabs | 米色凹槽 + 白色拇指 + 瓷青 700 文字 |
| Dialog/Drawer | 桌面用 Dialog，移动用底部 Drawer（圆角 28px 顶） |
| Toast | 顶部右侧 → 改为顶部居中，墨 900 底白字，朱砂 hairline |
| Chip | 1px 边框 + 8% 同色填充，**四色**：青/朱/金/墨 |

### 4.2 业务组件

#### A. 模型 Logo 圆角方块（`.logo-dot`）
- 28px 圆角 8px，纯品牌色背景，2 字符白色字。
- 缩小至 22px / 16px 用于列表与表格。

#### B. 印章（`.seal`）
- 朱砂红 `#C8472F`、6px 圆角、`rotate(-4deg)`、思源宋体加粗。
- 必有 inset 米白线 `inset 0 0 0 2px rgba(253,252,248,.35)` 模拟章身留白。
- 仅用于：登录意境区 / 万模卡 / 后台 logo 旁。

#### C. 万模卡
- 三色叠加渐变 `#1F6B66 → #103D3B → #1A1C20`。
- 暗纹 SVG `<pattern>` 祥云图案 + 朱砂印章。
- 数字必用 JetBrains Mono，超大字号 (40–48px) `font-weight: 300`。

#### D. 墨线分隔（`.ink-stroke`）
- 1px 渐变线 `linear-gradient(90deg, transparent, #2C2E33 20%, #2C2E33 80%, transparent)`。
- 用于分组、卡片内部分割，替代传统 hairline。

#### E. 远山 SVG 装饰
- 三层 `<path>`，颜色由浅至深：`#A5CCC4 → #5BA5A0 → #1F6B66`。
- 用于登录、绘图 Hero、空状态。

#### F. 中文标签样式（`.label-zh`）
- 思源宋体 + `letter-spacing: 0.25em` + 10–12px。
- 中间加空格 `下 午 好`，营造书法标牌感。

#### G. 流式输出指示
- 1.5×16px 瓷青 500 竖条，`animate-pulse`，替代圆点。

## 五、页面级规范

### 5.1 登录页 `/login`

**结构**：
- ≥1024px：左 1.1fr 意境（远山 SVG + 朱砂印章 + 品牌字） + 右 1fr 登录卡
- <1024px：顶部 banner（缩小意境） + 全宽登录卡

**关键规则**：
- Tab 顺序：**微信 → 账号 → 扫码 → 验证码**，按当前环境智能高亮第一项
- 微信按钮使用官方绿 `#07C160`，不要套品牌色
- 服务条款链接强制可点，文字 11px 居中、墨 400
- 移动端弱网降级：3 秒未拿到能力清单 → 直接显示账号密码 + 手机号

### 5.2 模型市集（首页/对话 Landing） `/chat`

**结构**：260 侧栏 + 主区。空对话时显示模型市集。

**关键规则**：
- 顶部"下午好 / 晚上好"按时间问候 + 用户名 + 额度 chip
- starter prompts 横向滚动，6–8 个，移动端必须可滑动
- 厂商分组用 `.label-zh` 标签 + `.ink-stroke` 分隔
- **国内模型置顶，海外其次** — 符合国内用户首选直觉
- 旗舰模型仅用 `ring-1 ring-celadon-200` 强调，不上色彩

### 5.3 对话页 `/chat/:id`

**消息气泡**：
- 用户：瓷青 600 底 + 米白字，圆角 16px 但右上角 4px
- 助手：白底 + 1px 墨色描边，圆角 16px 但左上角 4px
- 元信息（时间、token）置于气泡外，墨 400 + 10px

**输入区**：
- 28px 圆角白卡 + paper 阴影
- 左：附图/语音/命令，右：模型 chip + 字符计数 + 发送按钮
- 发送按钮 40×40 圆角 16px，瓷青渐变
- iOS 防缩放：`font-size: 16px` 起步

**模型切换**：
- 桌面：顶栏点击模型 chip → 下拉 Popover
- 移动：底部抽屉，支持搜索

### 5.4 绘图页 `/image`

**结构**：260 + flex + 320 三栏（桌面）

**关键规则**：
- 提示词框 3 行，下方一排"+ 国画风 / + 工笔 / + 水墨"chip 用于快速追加
- 参考图最多 4 张，缩略图 56×56 + 删除小角标
- 参数面板必须显式"内容审核"开关 + **《生成式人工智能服务管理暂行办法》提示语**（合规）
- 任务卡显示三态：完成 / 生成中（百分比 + 转圈）/ 失败（朱砂警示）
- Lightbox 在移动端必须支持双指缩放、单指拖动

### 5.5 个人中心 `/profile`

**关键规则**：
- 顶部"陆 朔"使用思源宋体大字，营造仪式感
- 万模卡为视觉锚点：永远在第一屏可见
- 三个快捷入口（看视频 / 兑换码 / 充值）紧贴卡下方
- 流水表头用 `.label-zh` 样式，行高 48px 留出呼吸
- 收入用瓷青 700、支出用朱砂 500，数字必用 mono 字体

### 5.6 后台 `/admin`

**关键规则**：
- 侧栏 `#0E0F11` 深墨色，与用户端纸色形成强烈对照
- KPI 卡：标签 `.label-zh` + 巨字 mono + 趋势 sparkline
- 图表三色固定：瓷青 500 / 鎏金 500 / 朱砂 500
- 表格 hover 米色，**绝不使用 zebra stripe**（中式审美不喜欢条纹）
- 移动端只保留**只读概览** + 最新请求列表，所有写操作（改套餐、发码）禁用

## 六、移动端补充规则

1. **底部 Tab 4 项**：对话 / 绘图 / 奖励 / 我的 —— `64px` 高（含安全区），图标 20px + 标签 10px。
2. **触控目标**：最小 44×44px（iOS HIG），删除按钮在 mobile 必须常显（不能 hover-reveal）。
3. **安全区**：`pt-safe / pb-safe / pl-safe / pr-safe` 全启用，刘海与底部条不遮挡内容。
4. **加载骨架**：用米色 + 月白 shimmer，**禁用紫色或蓝色** shimmer。
5. **页面切换**：右滑返回手势必须保留（Next.js App Router + `view-transition-name`）。
6. **输入框**：所有 `input/textarea` 字号 ≥16px，否则 iOS Safari 会自动缩放。

---

## 七、可访问性

- 所有按钮 `aria-label` 必填；图标按钮单独提供。
- 主 CTA 文本对比度 ≥ 4.5（瓷青 600 + 月白已达 6.2）。
- 焦点态：`outline: 2px solid #2F8C86; outline-offset: 2px;` — 不用蓝光默认。
- 流式输出区域加 `aria-live="polite"`。
- 印章是装饰图，`aria-hidden="true"`。
