# Chatty / 万模AI 小程序 PRD

版本：v1.1
日期：2026-05-04

## 1. 一句话说明

小程序只做一件核心事情：**用户观看激励视频广告获取 AI Token**。

它最多展示用户当前 token 余额、今日剩余领取次数和单次奖励额度，不做完整账户中心。

## 2. 产品定位

小程序定位：**微信生态里的广告领 Token 工具页**。

用户打开后只需要看到：

- 当前 token 余额。
- 今日还能看广告领取几次。
- 每次广告可获得多少 token。
- 一个主要按钮：`看广告领取 Token`。

## 3. 明确不做

v1.1 小程序不做：

- Web 扫码登录确认页。
- 小程序内 AI 聊天。
- 兑换码输入。
- token 流水列表。
- 个人中心。
- 使用规则长页。
- 客服页。
- 用户协议独立页。
- 复杂任务中心。
- 邀请奖励。
- 会员体系。

这些能力放在 Web 端或后续版本，不进入当前小程序范围。

## 4. 必须保留的技术能力

即使功能极简，后端仍然需要识别用户身份，因此保留：

- `wx.login`
- 后端 `code2Session`
- 小程序 session token
- token 余额查询
- 广告奖励配置查询
- 广告 reward session 创建
- 广告奖励 claim

如果用户未绑定 Chatty 用户：

- 小程序不做复杂绑定页。
- 首页提示：`请先在网页版完成微信登录后再领取 Token`。
- 领取按钮不可点击。

## 5. 页面结构

只注册两个页面：

- `pages/index/index`：首页，展示余额和广告领取。
- `pages/error/error`：错误兜底页。

其他页面不注册、不展示、不作为 v1.1 范围。

## 6. 首页内容

首页必须展示：

- 品牌：`万模AI`
- 副标题：`看广告领取 AI Token`
- 当前 token：例如 `123456`
- 今日剩余领取次数：例如 `今日还可领取 3 次`
- 单次奖励：例如 `+1000 tokens / 次`
- 主按钮：`看广告领取 Token`
- 简短规则：
  - 完整观看广告后发放。
  - 中途关闭广告不发放。
  - token 余额以服务器为准。

## 7. 广告领取流程

1. 小程序启动后调用 `wx.login`。
2. 后端换取小程序 session。
3. 首页请求用户余额和广告配置。
4. 用户点击 `看广告领取 Token`。
5. 小程序请求后端创建 reward session。
6. 后端返回 `rewardSessionId` 和 `adUnitId`。
7. 小程序播放激励视频广告。
8. 广告完整观看后，小程序调用 claim 接口。
9. 后端校验通过后增加 token。
10. 小程序弹窗显示领取成功。
11. 首页刷新余额和剩余次数。

## 8. 异常规则

- 用户未绑定：提示 `请先在网页版完成微信登录后再领取 Token`。
- 今日领完：按钮显示 `今日已领完`，不可点击。
- 活动关闭：按钮显示 `活动暂未开放`，不可点击。
- 广告加载失败：提示 `广告加载失败，请稍后重试`。
- 中途关闭广告：提示 `完整观看后才能领取`。
- 后端 claim 失败：提示 `领取失败，请稍后重试`。

## 9. 接口清单

小程序只使用这些接口：

- `POST /api/wechat/miniapp/auth/login`
- `GET /api/wechat/miniapp/me`
- `GET /api/wechat/miniapp/rewards/config`
- `POST /api/wechat/miniapp/rewards/sessions`
- `POST /api/wechat/miniapp/rewards/claim`

不使用兑换码、流水、扫码确认、客服、帮助、会员等接口。

## 10. 验收标准

- 小程序打开后能自动登录微信。
- 首页能展示 token 余额。
- 首页能展示今日剩余领取次数。
- 首页能展示单次奖励 token 数。
- 未绑定用户不能领取，并提示先在网页版完成微信登录。
- 点击按钮能播放激励视频广告。
- 完整观看后 token 增加。
- 中途关闭广告不增加 token。
- 今日次数用完后按钮不可点击。
- 活动关闭后按钮不可点击。
- 页面没有兑换码、流水、个人中心、客服、帮助、Web 登录确认等入口。
