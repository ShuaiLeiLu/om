# Chatty / 万模AI 小程序 PRD

版本：v1.1  
日期：2026-05-04

## 1. 一句话说明

小程序只做一件核心事情：**用户观看激励视频广告获取 AI Token**。最多再展示用户当前 token 余额和今日剩余领取次数。

小程序不是完整账户中心，不负责复杂登录确认、兑换码、流水、客服、帮助、会员、聊天等功能。

## 2. 产品定位

小程序定位：**广告领 Token 工具页**。

用户打开后只需要看到：

- 当前 token 余额。
- 今日还能看广告领取几次。
- 每次广告可获得多少 token。
- 一个主要按钮：`看广告领取 Token`。

## 3. 明确不做

v1.1 小程序不做：

- Web 扫码登录确认页。
- 小程序内 AI 聊天。
- 兑换码输入。
- token 流水列表。
- 个人中心。
- 使用规则长页。
- 客服页。
- 用户协议独立页。
- 复杂任务中心。
- 邀请奖励。
- 会员体系。

这些可以留给 Web 或后续版本。

## 4. 必须保留的技术能力

即使小程序功能极简，仍然需要后端识别用户是谁，因此保留：

- `wx.login`
- 后端 `code2Session`
- 小程序 session token
- 广告奖励接口
- token 余额查询

如果用户未绑定 Chatty 用户：

- 小程序不做复杂绑定页。
- 直接提示：`请先在网页版完成微信登录后再领取 Token`。

## 5. 页面结构

只注册两个页面：

- `pages/index/index`：首页，展示余额和广告领取。
- `pages/error/error`：错误兜底页。

其他页面不注册、不展示、不作为 v1.1 范围。

## 6. 首页内容

首页必须展示：

- 品牌：`万模AI`
- 副标题：`看广告领取 AI Token`
- 当前 token：例如 `123456`
- 今日剩余领取次数：例如 `今日还可领取 3 次`
- 单次奖励：例如 `+1000 tokens / 次`
- 主按钮：`看广告领取 Token`
- 简短规则：
  - 完整观看广告后发放。
  - 中途关闭广告不发放。
  - token 余额以服务器为准。

## 7. 广告领取流程

1. 小程序启动后调用 `wx.login`。
2. 后端换取小程序 session。
3. 首页请求用户余额和广告配置。
4. 用户点击 `看广告领取 Token`。
5. 小程序请求后端创建 reward session。
6. 后端返回 `rewardSessionId` 和 `adUnitId`。
7. 小程序播放激励视频广告。
8. 广告完整观看后，小程序调用 claim 接口。
9. 后端校验通过后增加 token。
10. 小程序弹窗显示领取成功。
11. 首页刷新余额和剩余次数。

## 8. 异常规则

- 用户未绑定：提示 `请先在网页版完成微信登录`。
- 今日领完：按钮显示 `今日已领完`，不可点击。
- 活动关闭：按钮不可点击，提示 `活动暂未开放`。
- 广告加载失败：提示 `广告加载失败，请稍后重试`。
- 中途关闭广告：提示 `完整观看后才能领取`。
- 后端 claim 失败：提示 `领取失败，请稍后重试`。

## 9. 接口清单

小程序只使用这些接口：

- `POST /api/wechat/miniapp/auth/login`
- `GET /api/wechat/miniapp/me`
- `GET /api/wechat/miniapp/rewards/config`
- `POST /api/wechat/miniapp/rewards/sessions`
- `POST /api/wechat/miniapp/rewards/claim`

不使用兑换码、流水、扫码确认、客服等接口。

## 19. 广告奖励防刷

### 19.1 小程序端必须做

- 未拿到 rewardSessionId 不播放广告。
- 只有 `isEnded === true` 才调用 claim。
- claim 过程中按钮置灰。
- claim 成功后刷新余额。
- 今日次数用完后不再播放广告。

### 19.2 后端必须做

小程序端判断不能作为唯一依据。后端必须：

- 校验用户是否登录。
- 校验用户是否绑定。
- 校验 reward session 是否存在。
- 校验 reward session 是否过期。
- 校验 reward session 是否已使用。
- 校验今日次数。
- 校验最小领取间隔。
- 幂等处理重复 claim。
